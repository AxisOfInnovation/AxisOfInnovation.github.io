tilt

What is tilt?

tilt is a library of commonly used functions our team has created throught its lifespan. These methods tend to be robot-independent working on any robot setup unless otherwise specified, so any other team may feel free to use them if they prove beneficial to their team.
No tilt doesn't mean anything. It's just a name. It's a joke, actually. Because an axis is a tilt.

How do I use tilt?

Using tilt is relatively simple as it is no different than just referencing files in your pre-existing programs. There are a few steps you'll have to run through to get it set up though.

  1. Download the zip file containing the sources to the project
  2. Unzip the project into the directory where your RobotC sources are
  3. When you need to use a file, put this statement at the top of your file
    #include "path/to/file.h"

    NOTE: The path is the path relative to the current file (the onethe include statement is written in)!

    NOTE: Some files (e.g. Infrared.h) require some more setup. If the compiler is showing an error in one of the tilt files, then check the page for the file to see if it requiers additional setup!

  4. Voila! You now have access to all the methods, tasks, and fields in the included file!

    NOTE: You'll have to include different files to access different methods,tasks, and fields!
    Be sure to check the page's documentation to see what is being included!

Why should my team use tilt?

Using code written by someone else is extremely common in programming, by using the RobotC functions you're already doing it, and there's no point in going through the arduous process of writing, testing, and debugging code if you know someone else has already (or is currently) doing it for you! We have found these files extremely useful, so hopefully you will too!
