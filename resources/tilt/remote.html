Remote.h

File Description

This header file is used to fetch controller updates on a seperate thread and allows easier access to controller values than its native counterparts. If a lost connection is detected between the Field Control System and the Samantha Module, then all controller input will be erased until the connection is regained. This means that if there is a problem with the connection the robot will not continue acting as it did before the connection was dropped; however, it will act as if no controller input is being provided at all.

All controller information is stored into publicly accessible data members, either an array or an int, seperated by controllers and input types.

Only members which are meant to be accessed, called, or started in the cases of fields, methods, and tasks, respectively, are shown, some members may be excluded.

Task Overview

task getJoystickUpdates()
Background task to get joystick information updates.
Starts the background task monitorPackets and then goes into an infinite loop which will wait for a packet from the Field Control System to be received, update the value of lastPacket to match the current system time, and then update all the buttons and joysticks to match the data the received packet contains.

task monitorPackets()
Monitors the time between packets from the Field Control System.
Continuously runs in the background, checking to see if the last packet received was less than one second ago. If a second passes before another packet is recieved, the getJoystickUpdates task will be stopped and all controller input values reset to false, 0, or DPAD_NONE, respective the their data types. When the next packet is received, getJoystickUpdates is restarted and controller input begins again.While this is listed here, it is not meant to be started directly, and is only listed so that you may understand fully what the relation between this and getJoystickUpdates is.
